04
Access & sync rules
How the team works without re-briefing Claude every session
Step 1 · Set up Claude Project
Create one Claude Project called "BridgeApp × Future Club — Phase 2". Upload: this Directory, the Master OS HTML, the Phase 2 PDF, and the Client Report DOCX. Every future session inherits this context. No more re-briefing.
Setup: DD · Frequency: Once
Step 2 · Mirror to Google Drive
Create the folder structure above in shared Drive. Set "BridgeApp_Phase2_OS" → "Anyone with link can view". Team bookmarks the Drive link. Files stay in sync because they live in one place.
Setup: DD · Frequency: Once
Step 3 · Push to GitHub → auto-deploy Netlify
All HTML files live in a GitHub repo. Netlify is connected to the repo — every commit auto-deploys. To update: download the new file from Claude → replace the file in your GitHub repo → commit with a clear message (e.g. "Update: Roboto typography") → Netlify deploys automatically. URL never changes.
Setup: DD · Frequency: When OS changes
Step 4 · Notion embed (optional)
In Notion: /embed → paste Netlify URL → resize full width. OS now lives inside Notion alongside the team's CRM, content calendar, and SOPs. One workspace, one truth.
Setup: Nono · Frequency: Once
Step 5 · Update workflow
Open the Claude Project → say what changed (e.g. "FC pod added at St Mary's" or "Update budget Engine 03"). Claude returns updated file. Download → replace in GitHub repo → commit → Netlify auto-deploys. URL stable. Team sees update immediately.
Owner: DD · Frequency: As needed
Step 6 · Live FC data automation
Make.com pulls BridgeApp backend → Google Sheets → Master OS dashboard updates automatically. Until live (Sep 2026 milestone), Nono updates the FC tracker manually every Friday via the dashboard "Update" form.
Owner: DD + Nono · Live: Sep 2026

Typography
Display: Roboto Black (900). Used for headlines, numbers, hero copy — breathable letter-spacing.
Body: Roboto Bold/Regular (700/400). Used for paragraphs, captions, UI.
Logo lockup rules
Pathfinder = Blue lockup. UniApply = Orange lockup. Future Club = Yellow lockup. Never mix tier colours within a single product callout.
Tone of voice
Direct. Confident. South African. Never patronising. No tech jargon. Speaks WITH students, not AT them. Says "Future Club" — never "the club" or "FC" in customer-facing copy.
Sales-first rule
Every output has one CTA: sign up · subscribe · register school · download app · book consult. Brand without sale = back to the brief.
